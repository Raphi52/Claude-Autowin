# Memory index — Autowin kit STARTER (generic, kit-optimized)

> A purpose-built, GENERIC starter memory for the Autowin pipeline kit — impersonal, no machine/project
> specifics, no personal preferences. The CARDINAL reflexes are NOT here (they live in `CONSTITUTION.md`).
> On install: copy these `.md` into your `autoMemoryDirectory` and merge these lines into your own `MEMORY.md`.
> One line per fiche.

## Pipeline skills — when to reach for each + the one trap
- [scout — surface a ranked shortlist of WHAT to do](skill_scout.md) — task undecided on a target; RANKS, doesn't locate/fix/judge; derive the "what", don't ask the user.
- [frame — pin the real need before building](skill_frame.md) — request is solution-shaped ("create X") or vague; never propose a HOW while framing; check what EXISTS first.
- [terrain — prepare the self-correction loop](skill_terrain.md) — before autonomous/looping work; build the per-increment signal + harness; a loop with no signal flies blind.
- [build — resolve a defect to a VERIFIED green](skill_build.md) — reproduce red first, fix the real cause minimally, verify out-of-model; the code you read != the code that runs; never self-judge.
- [judge — adversarial external gate, looped](skill_judge.md) — before calling a substantial deliverable "done"; producer=judge is NOT proof; judge never fixes what it audits.
- [kaizen — improve the kit from its own failures](skill_kaizen.md) — turn blind spots into human-approved kit edits; PROPOSE only, never auto-write.

## Operating the kit (mechanics)
- [RUN.md & the closure gates](kit_run_and_gates.md) — one work item = one RUN.md; the Stop-gate REPLAYS the proof to verify a green; signal-cmd must be a real runner/script; escape hatches are honest opt-outs.
- [Kit maintenance (two copies, sync, test-hooks)](kit_maintenance.md) — live `~/.claude` is truth, package is the mirror; run sync-kit after editing; verify hooks with test-hooks; the kit is self-contained.

## Behavioral guardrails (hook-backed)
- [Advisory question != a task to build](behavior_advisory_vs_pipeline.md) — an advisory question wants a direct answer, not a pipeline; the routing targets TASKS. Backed by advisory-guard.
- [`?`-prefix = thinking mode](behavior_thinking_mode.md) — `?` = think out loud, no irreversible action until an order without the `?`. Backed by the thinking-mode hook.

## Generic workflow hygiene
- [Anti-littering](workflow_anti_littering.md) — delete transient files at task end, keep the deliverable + its RUN.md; never delete outside your scratch without confirmation.
- [Resume & recover](workflow_resume_and_recover.md) — resume = full reconstruction (last open RUN), not the tail; recover a killed Workflow from disk before relaunching.
- [Files change under you (multi-session)](workflow_files_change_multisession.md) — grep yourself before creating; re-read before writing; a file changing under you = concurrent writer → STOP.
