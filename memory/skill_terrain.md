**Use when** — about to run Claude solo / looping to completion; "how will Claude get its screenshots-logs", "set up the autonomous work environment / self-correction", especially on a project where that harness does NOT exist yet.

**You get** — the loop spec the executor needs (per-increment signal that reproduces the user's symptom, decompose map, cost caps, green-checkpoint, need->loop->judge wiring) + the missing observability harness actually BUILT.

**Trap** — terrain prepares the HOW-to-loop; it does NOT frame the need (-> [[skill_frame]]) nor execute the loop mechanics (-> ENGINE Ch.4 BUILD, the executor's manual — no skill fires during the build itself). Build the per-increment SIGNAL first: an autonomous loop with nothing real to observe can only pretend to self-correct. Chains frame -> terrain -> [[skill_build]] -> [[skill_judge]].
