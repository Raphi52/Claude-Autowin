**Use when** — "create/add/make/set up X", writing a doc/README/config (check it doesn't already exist FIRST), a vague-but-chosen need, or "which approach / compare options / help me decide between designs".

**You get** — `## Besoin` written in the RUN.md (the real problem — not the requested artifact — scope in/out, a VERIFIABLE success criterion, stated assumptions, blind spots, risks) + (ONLY if the approach is open) `## Options` (>=3 scored, genuinely distinct) + a `Décision:` line.

**Trap** — do NOT propose a HOW during framing: the instant a solution is on the table you flip from PRODUCING (you drive) to REACTING (you defend an artifact). Trace "create X" back to the real problem (trap #1 = the duplicate -> check what EXISTS). A still-open premise under solution-phrasing -> stay conversational, converge the form WITH the user first; never assert "you confirmed X". -> hands off to [[skill_terrain]] (or straight to [[skill_build]]).
