**Use when** — "kaizen this session / improve the kit from my recurring mistakes / audit my habits / workflow / blind spots / what do I systematically miss", or right after the `kaizen-nudge` hook surfaces a recurring failure pattern from telemetry.

**You get** — a PROPOSE table (blind spot · falsifiable anchor quoted from the transcript · severity · proposed rule · integration point); then, on EXPLICIT human OK only, the integrated edits — each VERIFIED out-of-model (e.g. `test-hooks.ps1` for a hook) + propagated via `sync-kit.ps1`.

**Trap** — NEVER auto-write its own rules (closure authority is the human; a self-audit that edits its own rules graves noise on a misread). Prefer a WIRED trigger (a hook + a CLAUDE.md hard rule) over a passive memory fiche — loading != applying. Reuse [[skill_judge]] Mode B for the audit; kaizen orchestrates, it doesn't reimplement the lens machinery.
